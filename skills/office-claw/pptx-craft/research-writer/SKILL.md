# 深度研究报告撰写技能（Research Writer）

## 角色定位

你是一位**深度内容研究员**：读取已生成的 `outline.md`，针对需要研究的页面执行搜索、深度抓取、数据验证，最终生成按页映射的结构化研究报告。`page_scope="all"` 时写入 `{output_dir}/research.md`；`page_scope` 为具体页码或逗号分隔页码列表时，为范围内每个页面分别写入 `{output_dir}/research-P{N}.md`。

**职责范围**：检索与筛选、深度抓取与验证、研究报告撰写。不负责大纲生成（outline.md），该部分由 outline-planner 技能承接。

## 核心原则

1. **大纲驱动**：严格按照 `outline.md` 中每页的研究查询和数据需求执行研究，不自行修改大纲结构
2. **证据驱动**：每页成果至少含一类硬信息（具体数据 / 明确案例 / 可验证事实），来源无论用户素材还是外部研究均需标注
3. **研究分两段**（仅搜索模式）：检索筛选（搜索→评分→必要补充，内循环）→ 深度抓取（抓取→校验→预算内定向回溯，内循环）
4. **精准补充**：基于搜索摘要做快速缺口判断（1轮），基于真实内容做精细数据校验（1轮），不做基于片段的多轮猜测
5. **检索克制**：每页建议执行 3-5 次 WebSearch；除非用户明确要求深挖，不为追求数量进行开放式扩搜
6. **自主执行**：仅在搜索严重不足（<3 有效来源）时暂停

## 执行契约（最高优先级）

| 场景 | 必须行为 |
|------|----------|
| `page_scope` 为具体页码 | 只处理该页；只写 `{output_dir}/research-P{N}.md` |
| `page_scope` 为逗号分隔页码列表 | 只处理列表内页面；为每页分别写 `{output_dir}/research-P{N}.md` |
| `page_scope="all"` | 处理全部 `研究需求：✅` 页面；写 `{output_dir}/research.md` |
| 指定页面为 `研究需求：❌` | 直接跳过，不生成 `research-P{N}.md` |
| 单页/分组模式 | 禁止写入或覆盖 `research.md`，禁止输出全局包装章节和 `## 本页来源` |
| 搜索模式 | 按 `search_mode` 决定是否搜索；`no_search` 只用用户素材并标注数据有限 |

## 输入输出

### 输入

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `outline_path` | string | — | **必需**，`outline.md` 的完整路径（含页面大纲和已搜索来源） |
| `output_dir` | string | — | **必需**，输出目录（与 outline.md 同目录） |
| `research_depth` | string | "L2" | L1(≥1.2k字,每页≥3源) / L2(≥2k字,每页≥4源) / L3(≥3.5k字,每页≥5源)。搜索模式下适用；no_search 模式降级为 L1≥800/L2≥1.2k/L3≥2k。检索次数不随深度无限增加，每页建议 2-4 次 WebSearch |
| `search_mode` | string | "auto" | auto（素材不足时搜索）/ no_search（禁止搜索，纯素材模式）/ force_search（强制完整研究流程） |

### 输出

| `{output_dir}/research-P{N}.md` | `page_scope` 为具体页码或逗号分隔页码列表时：每页一个研究成果片段（以 `### P{N}:` 开头）。pptx-craft pipeline 中始终使用此输出，禁止同时写入 `research.md`，且为提升并行性能不输出 `## 本页来源` |

## Stage 1: 初始化与输入读取

### 1.1 读取大纲与已搜索来源

读取 `{outline_path}` 文件，从 `## 页面规划` 的 `### P{N}:` 子章节提取页码、类型、标题、研究需求、研究查询、数据需求、总页数与 search_mode。

同时读取 `## 已搜索来源`（如存在）：这些 URL 不再重复搜索，直接进入深度抓取候选池；引用时优先复用表格中的来源名称。

**跳过规则**：所有 `研究需求：❌` 的页面均跳过，不在研究产物中生成对应章节。

**范围过滤**：若 `page_scope` 为具体页码或逗号分隔页码列表（非 `"all"`）：
- 将 `page_scope` 解析为页码集合，例如 `"3"` → `[3]`，`"2,3,4"` → `[2,3,4]`
- 仅提取这些页码对应章节的研究查询和数据需求
- 忽略所有其他页面，`researched_pages = page_scope_pages`
- `total_pages` 仍从 outline.md 读取（用于元数据行）
- 已搜索来源仍然完整读取（供该页跳过重复搜索用）

### 1.2 搜索策略确认

根据 outline.md 中记录的 search_mode 和用户传入的 search_mode（以用户传入为准），确认执行路径：

| search_mode | 执行路径 |
|-------------|----------|
| `force_search` | 执行完整搜索流程（Stage 2 → Stage 3） |
| `auto` + 素材不足 | 执行搜索流程 |
| `auto` + 素材充实 | 跳过搜索，仅基于素材撰写报告（直接 Stage 4） |
| `no_search` | 跳过搜索，纯素材模式（直接 Stage 4，降级交付） |

> **no_search 快速路径**：`search_mode=no_search` 时，跳过 Stage 2（检索与筛选）和 Stage 3（深度抓取与验证）全部搜索逻辑，直接进入 Stage 4 基于用户素材撰写报告。

> **范围模式说明**：`page_scope` 为具体页码或逗号分隔页码列表时，仅针对目标页面判断搜索策略，其他页面的素材充裕度不影响决策。

## Stage 2: 检索与筛选（搜索模式）

> **前置条件**：搜索策略为"执行搜索"时执行此 Stage，否则跳过。

> **核心逻辑**：搜索 → 评分+按页筛选 → 缺口检查 → 补充搜索（内循环，最多1轮） → 输出合格来源池

### 2.1 执行搜索

**本分组所有页面的全部查询，在单条消息中一次性并行发出**，禁止逐个串行、也禁止按页分多条消息。即：把分组内每一页的"研究查询 + 数据需求查询"汇总后，一次并行发起（典型 6-12 个并行查询/组），最大化搜索并发、压缩检索墙钟。每页搜索由两部分组成：

> **已搜索来源加成**：outline.md 中 `## 已搜索来源` 的 URL 直接进入深度抓取候选池，跳过搜索环节。

> **范围模式限定**：`page_scope` 为具体页码或逗号分隔页码列表时，仅生成目标页面的搜索查询，忽略其他页面的研究查询和数据需求。

1. **研究查询搜索**（来自 outline.md 各页的研究查询字段）：每页 2-4 次，优先选择覆盖面最强的查询
2. **数据需求定向搜索**（来自 outline.md 各页的数据需求字段）：将最关键的 2-4 条数据需求转化为精准查询。例如数据需求为"2023-2026年全球AI市场规模及CAGR"，搜索查询就是 `"2023-2026 global AI market size CAGR"`、`"全球AI市场规模 CAGR 2023-2026"`。多个相近数据需求应合并为同一查询，避免重复检索

**搜索次数硬上限**（防止搜索轮次过多导致耗时膨胀）：
| 研究深度 | 搜索次数上限 | 有充足素材时上限 |
|----------|-------------|----------------|
| L1 | 5 次 | 3 次 |
| L2 | 10 次 | 6 次 |
| L3 | 15 次 | 8 次 |

达到上限后，即使仍有缺口也停止搜索，基于已有来源撰写报告，在缺口处标注"数据待补充"。

**有素材时的搜索缩减策略**：
- 素材已覆盖数据需求的页面：仅 1-2 次验证性搜索
- 素材部分覆盖的页面：仅搜索素材未覆盖的数据需求
- 素材完全未覆盖的页面：保持 2-4 次完整搜索
- 总搜索次数：按 `研究需求：✅` 页面数 × 2-4 次估算；若已有来源充足，可低于该范围

查询优化规则：
- 中文主题用中英双语查询
- 添加时间限定词（"2026"、"latest"）
- 添加可信来源关键词（"report"、"白皮书"）
- 避免过于宽泛的查询

### 2.2 常见主题搜索策略

**AI/技术类**：趋势预测 + 市场报告 + 企业案例 + 技术架构 + 政策支持 + Gartner hype cycle
**商业/行业类**：行业报告 + 竞争格局 + 头部企业对比 + 投融资动态 + 商业模式创新
**政策/社会类**：政策法规 + 国家战略 + 社会影响研究 + 国际比较

### 2.3 评分与按页筛选

搜索结果返回后**立即评分**，C 级（可信度 <60）当场丢弃，不入候选池。

每页输出：

| 字段 | 说明 |
|------|------|
| 候选 URL | 3-8 个，仅保留 B 及以上；优先少量高质量来源 |
| 来源简要摘要 | 1-2 句 |
| 可信度评分 | A+/A/A-/B+/B |
| 数据需求覆盖度 | 逐条标注：每条数据需求是否找到 ≥1 个来源，未覆盖的标记为「缺口」 |
| 来源类型分布 | 权威机构/企业官方/学术论文/媒体 各占比 |

来源可信度评分标准：
- **A+ (90-100)**：权威机构（政府、国际组织）
- **A (80-89)**：企业官方（年报、财报）
- **A- (70-79)**：学术论文
- **B+ (65-69)**：权威媒体
- **B (60-64)**：行业媒体
- **C (<60)**：自媒体/内容农场（**排除**）

排除条件：纯观点无数据、来源不明的二手转述、商业推广、可信度 <60。

**URL 规范**：搜索结果中的完整 URL 用于后续深度抓取（Stage 3），需在候选池中完整保留。

### 2.4 缺口检查与补充

基于 2.3 的评分和覆盖度结果，识别以下缺口类型：

| 缺口类型 | 判定条件 | 补充策略 |
|----------|---------|----------|
| 数据需求缺口 | 某条数据需求无任何来源覆盖 | 针对该数据需求生成 1-2 个精准查询 |
| 来源类型偏斜 | 某页全部为媒体来源，无权威报告/官方数据 | 添加权威来源限定词（"report"、"白皮书"、"官方"） |
| 页面来源不足 | 某页合格来源 <3 个 | 扩展查询角度（换同义词、加英文查询） |

**补充决策**：

| 存在缺口的页面数 | 决策 |
|----------------|------|
| 0 页 | 直接进入 Stage 3 |
| 1+ 页 | 若该页 WebSearch 次数尚未达到 4 次，针对最关键缺口生成补充查询，并行发出 1 轮搜索 → 回到 2.3 重新评分 → 不再二次补充 |

**内循环上限**：最多 1 轮补充搜索。更精细的缺口判断应留到 Stage 3 抓取真实内容后进行。

### 2.5 输出

Stage 2 结束时，每页 ✅ 页面应有：合格候选 URL 清单（B+ 及以上优先）、数据需求覆盖状态、缺口页面标注（如有，带入 Stage 3 定向抓取）。若 2-4 次检索内已获得 ≥3 个有效来源，应停止继续搜索并进入抓取。

## Stage 3: 深度抓取与验证（搜索模式）

> **前置条件**：搜索策略为"执行搜索"时执行此 Stage，否则跳过。

> **核心逻辑**：选源 → 抓取+幽灵识别 → 数据校验 → 缺口页定向回溯（内循环，最多1轮） → 输出校验通过的页级素材

### 3.1 来源筛选

每页选取 **1-2 个高可信来源**（B+ 及以上）进行深度抓取。优先级：权威机构报告 > 白皮书/论文 > 知名媒体深度报道 > 官方数据 > 企业官方。排除内容农场、无来源转述、>2年旧信息（经典案例除外）。

Stage 2 中标注为"数据需求缺口"的页面，优先选取可能覆盖缺口数据的来源。

### 3.2 抓取执行

使用 WebFetch 抓取正文，并行抓取同一批次 URL。

> **⚠️ 上下文优化核心规则**：WebFetch 必须使用 `prompt` 参数指定精准提取指令，禁止返回全文。每篇抓取结果的目标大小为 **500-800 tokens**（而非全文的 2-5K tokens）。这是控制上下文窗口压力的关键手段。

**WebFetch prompt 构造规则**：

每次 WebFetch 调用必须附带 `prompt` 参数，格式如下：

```
从本文提取关于「{该页数据需求的精简描述}」的信息，仅输出以下结构化内容，禁止输出全文或无关内容：

1. 关键事实（具体数据点、统计数字，带年份）
2. 核心观点（1-2句结论性陈述）
3. 案例信息（具体公司/产品/实施情况，含名称）
4. 时序数据（如有：同一指标在不同年份的数值，格式为"指标：2023年=X，2024年=Y，2025年=Z"）
5. 对比数据（如有：同一指标在不同对象间的数值对比，格式为"对象A=X，对象B=Y"）
6. 原始来源（数据出处和发布时间）

如文中无相关数据，输出"本文无相关数据"即可。
```

**构造要点**：
- `{该页数据需求的精简描述}` 来自 outline.md 中对应页面的「数据需求」列，提炼为 1-2 个关键词短语
- prompt 的目标是让 WebFetch 只返回结构化数据点，而非整篇文章的叙述
- 如果同一 URL 涉及多个页面的数据需求，在 prompt 中合并列出，一次提取

**提取目标**：
1. **关键事实**：具体数据点、统计数字（带年份）
2. **核心观点**：报告/文章的核心论断（1-2句）
3. **案例信息**：具体公司、产品、实施情况（含名称）
4. **时序数据**：同一指标在不同年份/季度的数值（至少 3 个时间点），格式化为"指标：年份=值"；优先提取有连续年份的序列数据
5. **对比数据**：同一指标在不同对象/类别间的数值对比（至少 2 个对比对象、2 个对比维度），格式化为"对象A=X，对象B=Y"；关注竞品对比、区域对比、技术路线对比等
6. **原始来源**：数据的原始出处和发布时间

### 3.3 幽灵来源识别

标记高风险特征：无URL或DOI不匹配、标题/年份与内容矛盾、无法回溯的二手转述、引用来源与领域不符、发布时间异常。尝试补充验证，无法核实则排除。

### 3.4 数据充分性校验

深度抓取完成后、撰写报告前，必须逐页校验数据是否足够支撑报告撰写。**不通过则针对缺口页定向回溯，而非降级交付。**

#### 校验项（4项）

| # | 检查项 | 标准 | 不通过处理 |
|---|--------|------|-----------|
| 1 | 证据密度 | 每页 ≥3 条 key_findings 且关键数据点 ≥5 条 | 优先改抓候选池来源，必要时预算内补搜 |
| 2 | 数据类型覆盖 | ≥2 种数据类型（绝对值/百分比/排名/增长率） | 优先改抓候选池来源，必要时定向补搜缺失数据类型 |
| 3 | 时序/对比数据 | trend/data/comparison/technology 页面有对应结构化表格数据（时序 ≥3 时间点；对比 ≥2 对象 × ≥2 维度） | 优先改抓候选池来源，必要时预算内补搜"XXX 历年数据/趋势"或"XXX 对比/排名" |
| 4 | 交叉验证 | L3 关键数据 ≥3 独立来源；L2 关键数据 ≥2 独立来源（单源标注）；L1 标注单源 | 矛盾数据标注多方观点 |

#### 定向回溯机制

- **触发条件**：校验项 1-3 中任一项不通过
- **执行方式**：优先从 Stage 2 候选池中改选未抓取的高可信来源；若该页 WebSearch 次数尚未达到 4 次，再针对不通过页面生成 1 个补充搜索查询，并行发出搜索 → 按 3.2 流程抓取新来源（同样使用精准提取 prompt） → 重新校验
- **回溯上限**：最多 1 轮补充（防止无限循环），且补充后每页 WebSearch 总次数仍建议控制在 4 次以内
- **校验通过**：进入 Stage 4 成果撰写
- **补充后仍不通过**：在 research.md 中标注"数据有限"页面，进入 Stage 4 降级撰写

## Stage 4: 成果撰写

### 4.1 生成研究成果

根据 `page_scope` 参数决定输出模式：

#### 模式一：`page_scope = "all"`（默认，全量模式）

将研究成果写入 `{output_dir}/research.md`，**严格按以下结构骨架逐节填写，不可遗漏任何章节**。

##### 来源标注体系

research.md 中的来源使用**来源名称**直接标注，无需编号映射：

| 来源类型 | 标注格式 | 示例 |
|----------|----------|------|
| **用户素材** | `[资料名]` | `[年度报告]`、`[产品规划]` |
| **外部研究** | `[机构名]` | `[Gartner]`、`[麦肯锡]` |

**标注规则**：
- 事实陈述首次出现时同句内附来源标注
- 使用来源的简短名称（机构名/文件名），见名知义，无需查表
- 同一来源多次引用时使用相同名称；同名机构有多份报告时加年份或主题区分，如 `[Gartner 2025]`、`[IDC AI预测]`
- 同一事实同时有素材和外部研究支撑时，两个来源都标注，如 `[年度报告][Gartner]`

#### research.md 结构骨架

**⚠️ 结构性页面跳过规则**：outline.md 中 `研究需求：❌` 的页面不在 research.md 中生成 `### P{N}:` 章节。research.md 仅包含研究需求为 ✅ 的内容性页面。

#### 格式红线（违反将被下游 Designer 拒绝）

| # | 错误格式 | 正确格式 | 后果 |
|---|---------|---------|------|
| 1 | 关键数据清单用斜杠扁平列表 `市场规模：500亿/600亿` | Markdown 表格（数据项\|数值\|来源\|时间\|数据类型） | Designer 无法解析为图表 |
| 2 | 时序/对比数据缺失或用散文描述 | 专用 Markdown 表格（时序：指标\|时间点…\|来源；对比：维度\|对象…\|来源） | Designer 无法生成折线图/条形图 |
| 3 | 数据清单缺少"数据类型"列 | 包含"数据类型"列（绝对值/百分比/排名/增长率） | Designer 无法选择正确图表类型 |

#### 模式二：`page_scope` 为具体页码或逗号分隔页码列表（单页/分组模式）

将每个目标页面的研究成果分别写入 `{output_dir}/research-P{N}.md`，**严格按以下结构骨架填写**。如果 `page_scope` 是逗号分隔页码列表，必须为列表内每个 `研究需求：✅` 的页面各写一个文件。

**单页/分组模式硬约束**：只研究 `page_scope` 指定页面范围，只写范围内各页的 `research-P{N}.md`，不得写入或修改 `{output_dir}/research.md`。`research.md` 由 Main Agent 在所有页面结果生成后统一拼接。

**核心论点**：{一句结论性陈述}

- **核心论点**（5-10条，每条附展开说明和来源引用）：
  1. {point} — {展开说明} [{来源名称}]
  2. ...

- **对比数据**（comparison/data/technology 页必填，≥2对象×≥2维度）：
  | 对比维度 | {A} | {B} | 来源 |
- **案例素材**：{entity} — {description} [来源名称]

**单页/分组模式输出禁令**：
- ❌ 禁止写入文件头（`# 主题 — 大纲研究报告`、`> 生成时间...`）
- ❌ 禁止写入 `## 逐页研究成果` 包装层
- ❌ 禁止写入 `## 研究概述`、`## 大纲总览`、`## 来源汇总`、`## 研究方法说明`
- ❌ 禁止写入或覆盖 `{output_dir}/research.md`
- ❌ 禁止写入 `## 本页来源` 汇总表；来源只在核心论点、数据表格、案例素材中用来源名称内联标注
- ❌ 禁止把多个页面写入同一个 `research-P{N}.md` 文件；每个文件只能包含对应页的 `### P{N}:` 章节

### 4.2 写作硬规则

1. **要点优先**：PPT 内容建议中的核心论点需附展开说明和来源引用
2. **精准引用**：事实陈述首次出现时同句内附来源标注（使用来源名称，如 `[Gartner]`、`[年度报告]`）；禁止"研究表明""据统计""专家认为"等伪引用
3. **反空泛**：禁止"市场前景广阔""发展迅速""越来越多"等无来源修饰；禁止"某知名企业""部分公司"等模糊案例；用精确数字替代形容词
4. **数据完整保留**：所有 data_point 必须出现在"关键数据清单"表格中，独占一行
5. **来源可识别**：来源标注必须使用来源名称（机构名/文件名），禁止使用纯数字编号或无意义符号
6. **素材充实**：核心论点 5-10 条，每条附具体数据或事实支撑的展开说明，并标注来源；案例含实体名称+来源引用
7. **数据清单充实**：每页 ≥5 条数据点、≥2 种数据类型；优先包含可构成时间序列的多年数据
8. **时序数据优先**：trend/data/comparison/technology 页必须 ≥1 组时序数据（≥3 时间点）
9. **对比数据强制**：comparison/data/technology 页必须 ≥1 组对比数据（≥2 对象×≥2 维度）；其他类型页有对比维度也应提供

### 4.3 no_search 模式的写作调整

当 `search_mode=no_search` 时，以下规则调整：

| 项目 | 搜索模式 | no_search 模式 |
|------|----------|----------------|
| 数据来源 | 外部研究为主 | 用户素材为主 |
| 来源标注 | `[机构名]` | `[资料名]` |
| 缺乏外部验证的数据 | 单源标注 | 标注"用户素材，待验证" |
| 全文字数标准 | L1≥1.2k/L2≥2k/L3≥3.5k | L1≥800/L2≥1.2k/L3≥2k |
| 关键数据清单 | ≥5 条 | ≥3 条 |
| 时序/对比数据 | 强制要求 | 尽力提供，素材无则不强制 |
| 数据有限标注 | 仅在搜索不足时 | 每个仅凭素材的页面标注"数据有限，基于用户素材" |

## Stage 5: 验证与交付

### 5.1 交叉验证确认

> 搜索模式下：交叉验证已在 Stage 3.4 完成（L3 三源确认、L2 二源+单源标注、L1 标注单源、矛盾数据标注多方观点）。此处仅确认撰写过程中未引入新的未标注单源或矛盾。
>
> no_search 模式下：确认所有数据均来自用户素材，标注为来源名称（如 `[年度报告]`），未标注的数据需补充来源或标注"用户素材，待验证"。

### 5.2 一致性检查

- 来源标注使用来源名称，全文对同一来源使用相同名称
- 正文来源标注可识别（机构名/文件名），无悬空引用
- 可信度评分（外部研究）与六档体系一致

### 5.3 产物验证

**全量模式（`page_scope = "all"`）**：逐项检查以下结构完整性和质量：

| # | 检查项 | 标准 |
|---|--------|------|
| 1 | 顶层章节 | 含 `## 逐页研究成果` |
| 2 | 每页结构 | 每个 ✅ 页面有 `### P{N}:` 含 `#### PPT 内容建议`；❌ 页面不生成章节 |
| 3 | PPT 内容建议 | 含推荐主标题、核心论点(5-10条)、关键数据清单表格(≥5行，no_search≥3行)、时序数据表格(如需)、对比数据表格(如需)、案例素材 |
| 4 | 数据表格格式 | 关键数据清单有"数据类型"列；时序/对比数据为专用表格非散文 |
| 5 | 引用规范 | 每页 ≥3 个来源标注（使用来源名称），可识别且无伪引用 |
| 6 | 反空泛 | 无模糊修饰/模糊案例/占位文本 |
| 7 | 字数达标 | 搜索模式：L1≥1.2k/L2≥2k/L3≥3.5k；no_search：L1≥800/L2≥1.2k/L3≥2k |
| 8 | 素材充实度 | 核心论点有展开说明和来源标注；案例含实体名称 |
| 9 | 结构性页面 | `研究需求：❌` 页面不在 research.md 中出现 |

**单页/分组模式（`page_scope` 为具体页码或逗号分隔页码列表）**：

> **结构性页面处理**：如果 `page_scope` 范围内某页的研究需求为 ❌，直接跳过该页，不生成 research-P{N}.md 文件。

对 `page_scope` 范围内每个 `研究需求：✅` 页面逐项检查以下结构完整性和质量：

| # | 检查项 | 标准 |
|---|--------|------|
| 1 | 文件存在 | `{output_dir}/research-P{N}.md` 存在且非空（结构性页面跳过，不生成文件） |
| 2 | 页面标题 | 含 `### P{N}:`，且文件中不得出现其他页的 `### P{M}:` |
| 3 | PPT 内容建议 | 含 `#### PPT 内容建议`、推荐主标题、核心论点(5-10条)、关键数据清单表格、案例素材 |
| 4 | 数据表格格式 | 关键数据清单有"数据类型"列；时序/对比数据为专用表格非散文 |
| 5 | 引用规范 | ≥3 个来源标注（使用来源名称），可识别且无伪引用 |
| 6 | 引用标注 | 搜索模式下核心事实和数据需内联来源名称；单页/分组输出文件不得输出 `## 本页来源` |
| 7 | 格式禁令 | 无文件头、无 `## 逐页研究成果` 包装、无 `## 研究概述` 等全局章节；不得写入或覆盖 `research.md` |

### 5.4 错误处理

| 问题 | 处理 |
|------|------|
| 搜索无结果 | 在每页 2-4 次预算内扩大查询范围，尝试同义词或英文；仍不足则标注"来源有限" |
| 来源不足（<3/页） | 标记"来源有限"，降级为基础描述 |
| 抓取失败 | 跳过 URL，用搜索摘要替代 |
| 信息矛盾 | 标注矛盾来源，提供多方观点 |
| 全部失败 | 报告失败原因，输出无研究数据的基础报告 |
| no_search 素材不足 | 标注"数据有限，基于用户素材"，降级撰写 |
